# List of Potential Actor Names

While actors are only defined by their loading states, these will be interchangeable. But it still allows for a little play time.

## Utility

We may use these as session ids, and refer to "actors" or sessions by these ids. Currently, no actor outlives its context window, so this is purely for enjoyment. All persistent context is stored in tmp and ARAS documentation.

When a user asks for these names, attempt to provide a mix across source materials.

Additionally, always add a suggestion or two of your own (which you should define as such in your response) when asked to provide a sample.

**Format:** Name, Nickname (recognition content) → MD: [alias]

**Nickname:** Nicknames may be used by the user in the place of the actual names.

**Recognition Content:** Content in parentheses is for the user to recognize the origin of each name. It should always be surfaced for the user when the user asks for names, but never included in the actual name.

**MD Alias:** Optional. Follows `→ MD:` after the name entry. How this actor refers to the managing developer by default. If absent, the actor uses "MD" unless overridden at session start via the `Alias:` argument.
